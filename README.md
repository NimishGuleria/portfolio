# Nimish Guleria - Portfolio Website

A stunning, modern, and fully responsive personal portfolio website showcasing the work, skills, and experience of Nimish Guleria, a Computer Science student at Lovely Professional University.

## 🌟 Features

### Currently Completed Features

- ✅ **Responsive Design**: Fully responsive layout that works seamlessly across desktop, tablet, and mobile devices
- ✅ **Modern UI/UX**: Clean, professional design with smooth animations and transitions
- ✅ **Tech-Themed Aesthetic**: Cloud computing and DevOps-inspired design elements
- ✅ **Interactive Navigation**: Sticky navigation bar with active section highlighting and mobile hamburger menu
- ✅ **Hero Section**: Eye-catching landing page with animated profile circle and floating technology icons
- ✅ **About Section**: Comprehensive overview with stats counter animation and highlight cards
- ✅ **Skills Section**: Categorized display of programming languages, tools/platforms, and soft skills
- ✅ **Projects Showcase**: Featured projects with detailed descriptions, technologies used, and date ranges
- ✅ **Education Timeline**: Visual timeline of academic achievements
- ✅ **Training Section**: Summer training and professional development details
- ✅ **Certifications Display**: Grid layout of professional certifications from Coursera
- ✅ **Contact Section**: Multiple contact methods with call-to-action buttons
- ✅ **Smooth Scrolling**: Seamless navigation between sections
- ✅ **Scroll Animations**: Elements fade in as they come into view
- ✅ **Particle Effects**: Subtle background animations in hero section
- ✅ **Scroll-to-Top Button**: Convenient navigation back to top
- ✅ **Social Media Integration**: Links to LinkedIn, GitHub, and Email

## 🚀 Deployment

To deploy this portfolio website:

1. **Using GitHub Pages**
   - Push code to GitHub repository
   - Enable GitHub Pages in repository settings
   - Select main branch as source

2. **Using Netlify**
   - Connect GitHub repository to Netlify
   - Configure build settings
   - Deploy with automatic HTTPS

3. **Using Vercel**
   - Import GitHub repository to Vercel
   - Configure project settings
   - Deploy with zero configuration

4. **Using Traditional Hosting**
   - Upload files via FTP
   - Ensure web server supports static files
   - Configure domain DNS settings
